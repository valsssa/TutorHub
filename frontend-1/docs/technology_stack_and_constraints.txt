
# Technology Stack & Constraints

## 1. Core Framework & Language
*   **Framework:** React 19
    *   Utilizes Functional Components and Hooks (`useState`, `useEffect`, `useMemo`, `useCallback`).
    *   Strict Mode enabled.
*   **Language:** TypeScript (ESNext target).
*   **Styling:** Tailwind CSS (v3.4 via CDN).
    *   Dark Mode strategy: `class` based.
    *   Custom color palette extension (Emerald/Slate).

## 2. State Management
*   **Strategy:** Centralized "Lifted State" + Local Component State.
*   **Store:** `App.tsx` serves as the primary store, holding:
    *   User session (`currentUser`).
    *   Domain data (`tutors`, `sessions`, `chatThreads`).
    *   UI state (`currentView`, `theme`, `modals`).
*   **Data Flow:** Unidirectional data flow (Parent -> Child via props).
*   **Persistence:** None (In-memory only). Data resets on browser refresh.

## 3. Routing Strategy
*   **Type:** Custom Conditional Rendering (Single Page Application).
*   **Implementation:** 
    *   Controlled by a `currentView` state variable (Type: `ViewState`).
    *   No integration with the Browser History API (Browser "Back" button does not navigate app views).
    *   No client-side routing library (like `react-router-dom`) is currently installed.

## 4. Build System & Module Loading
*   **Development/Runtime:** Native ES Modules via `<script type="importmap">`.
*   **Dependency Management:** CDN-based imports via `esm.sh`.
    *   React, ReactDOM, Lucide-React, Recharts, and Google GenAI SDK are loaded directly from the CDN.
*   **Transpilation:** Client-side Babel (implied by the environment) or Vite (recommended for production build).

## 5. Environment Separation
*   **Configuration:**
    *   **API Keys:** Managed via `process.env.API_KEY`.
    *   **Feature Flags:** None currently implemented.
*   **Environments:**
    *   **Development/Prototype:** Uses `constants.ts` for data mocking and `esm.sh` for dependencies.
    *   **Production (Future):** Would require a build step (Vite/Webpack) to bundle assets, strip mock data, and implement real API calls.

## 6. External Integrations
*   **AI:** Google GenAI SDK (`@google/genai`) accessing Gemini models (`gemini-3-flash-preview`).
*   **Icons:** Lucide React.
*   **Charts:** Recharts.

## 7. Constraints
*   **Browser Support:** Modern browsers supporting ES Modules and CSS Variables.
*   **Performance:** All data is loaded client-side; initial load time scales with the size of `constants.ts` and dependency network requests.
*   **Security:** 
    *   No authentication token storage (simulated login).
    *   API Key is exposed in the frontend source (acceptable for prototype/demo, requires proxy/backend for production).
